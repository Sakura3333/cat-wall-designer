**项目理解**
这是一个 Vite + React + TypeScript 前端原型，用 Zustand 管编辑器状态，用 Three.js / React Three Fiber / drei 做 3D 场景，用 dnd-kit 做组件拖拽。核心产品是：上传室内图或选择模板，基于透视标线/四点兼容流程生成厚墙体与地面，再把猫墙组件放入 3D 场景。

当前已完成的主线是：图片上传、透视标线、标尺、模板房间、厚墙/地面、3D 选择/移动/旋转、属性面板、组件 catalog、`/components_manager` 管理页、组件拖入后生成 box 占位。关键断点也很清楚：现在 [EditorPage.tsx](E:/code/PSD3/src/editor/EditorPage.tsx:17) drop 后只调用 `addComponent(kind)`，而 [editorStore.ts](E:/code/PSD3/src/editor/editorStore.ts:461) 只按组件大类找一个默认 plane 并给默认偏移位置；[SceneCanvas.tsx](E:/code/PSD3/src/features/scene3d/SceneCanvas.tsx:261) 仍用 `boxGeometry` 渲染组件，`TransformControls` 提交时也只是直接写回 transform。
